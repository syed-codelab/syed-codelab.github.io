Part A: Setup

Adding files and creating directories in MULE

The following steps require the creation of directories and adding of files at various locations in the MULE file system.

To create a directory/folder in MULE: First, make sure the File Manager app is open. To open the File Manager, click Menu on the MULE taskbar, then click Utilities > File Manager. In the File Manager, navigate to the desired location and then click File > Create new directory. Enter a title in the text-box and then click OK to create the folder.

There are two ways to add a file to a directory in MULE:

  1. Use the Textpad tool within MULE to create the file and then save it to the desired location. Open the Textpad via Menu > Utilities > Textpad, then add the required code and click File > Save As.
  2. Create the file on your own system using a text editor or IDE, and then drag and drop the file from your own system into the MULE File Manager open at the desired location.

Step 1: Setting up Workbook

When you first open the Workbook app in MULE, it might appear as a blank window. This is because it has not yet been initialised. The steps below outline how Workbook is initialised and the folder structure should be set up.

1.1 Navigate to course:/config/ and add a file config.json

Open File Manager in MULE and navigate to the config directory by clicking course > config. Here add a file named config.json.

The code for this file can be found in Part B.

1.2 Return to the course directory and create a directory called workbooks

In File Manager, return to the course directory and inside this directory create a directory called workbooks.

1.3 Add a workbooks.json file to course:/workbooks/

Inside the workbooks directory, create a file and name it workbooks.json.

The code for this file can be found in Part B.

1.4 Create a source directory at course:/workbooks/

Inside the workbooks directory, create a new directory and name it source.

1.5 Add files metadata.json and description.html to course:/workbooks/source/

Inside the source directory, create two files and name them:

  1. metadata.json
  2. description.html

Some example code for these files can be found in Part B.

Now that the setup is complete, reopen Workbook - the main page should be visible.

Step 2: Adding Labs

Recompiling the Workbook

When changes are made to the files and directories used for the function of the Workbook, it will need to be recompiled in order for these changes to be activated. To recompile the Workbook: open the Workbook via Menu > Other > Workbook. Then click Admin, expand the Compile Workbook section and click (RE)COMPILE. The Workbook must then be closed and reopened for the changes to take effect.

2.1 Create a directory Labs at course:/workbooks/source/

2.2 Add files metadata.json and description.html to course:/workbooks/source/Labs/

2.3 For each new lab, create a directory Lab_XX at course:/workbooks/Labs/

Replace XX with the lab number e.g. Lab_01.

Inside each Lab_XX directory, create two files and name them:

  1. metadata.json
  2. description.html

The code for metadata.json can be found in Part B.

Step 3: Adding Questions

3.1 For each new question in each Lab, create a directory qXX_title at course:/workbooks/source/Labs/Lab_YY

Replace XX with the question number and replace title with a descriptive word. YY refers to the lab number used in the previous step.

E.g. Lab_01/q02_HelloWorld/

Inside each qXX_title directory, create two files and name them:

  1. metadata.json
  2. description.html

Extra: Adding sample input/output tables to question descriptions

Most of the lab questions will require some sample inputs and outputs to be displayed to the student. These can be written in the question-level description.html document as a html table. There is some CSS boilerplate in Part B that can be copied to the top of each description.html document to ensure consistent readable styling across all questions.

A table in HTML is defined using the <table> tag. Inside this, the rows are defined using a <tr> tag. If the row represents a table header, we define each cell in the row with the th tag, or if the row is not a header, we define each cell using the <td> tag.

See some example HTML code below which produces a simple table:

<table>

    <!-- header -->
    <tr>
        <th>
            Sample input
        </th>
        <th>
            Sample output
        </th>
    </tr>
    <!-- /header -->

    <!-- sample -->
    <tr>
        <td>
            1
        </td>
        <td>
            10
        </td>
    </tr>
    <!-- /sample -->

</table>

This code will produce a table like so:

To add more rows (each row represents a sample input and output), just add another <tr> tag containing two <td> tags like so:

<table>

    <!-- header -->
    <tr>
        <th>
            Sample input
        </th>
        <th>
            Sample output
        </th>
    </tr>
    <!-- /header -->

    <!-- sample 1 -->
    <tr>
        <td>
            1
        </td>
        <td>
            10
        </td>
    </tr>
    <!-- /sample 1 -->

    <!-- sample 2 -->
    <tr>
        <td>
            20
        </td>
        <td>
            200
        </td>
    </tr>
    <!-- /sample 2 -->

</table>

These tables don’t look very appealing and are confusing to read. When we add the CSS boilerplate from Part B to the top of the document, a much more readable table is generated.

<!-- CSS boilerplate -->
<style>
    table {
        width: 100%;
        border-collapse:"separate";
        border-spacing: 4px 8px;
    }

    th {
        text-align: left;
    }

    td {
        background-color: #f5f5f5;
        border: 1px solid black;
        border-radius: 2px;
        vertical-align:top;
        padding: 4px 8px
    }

</style>
<!-- /CSS boilerplate -->

<!-- Sample IO table -->
<table>

    <!-- header -->
    <tr>
        <th>
            Sample input
        </th>
        <th>
            Sample output
        </th>
    </tr>
    <!-- /header -->

    <!-- sample *duplicate this for more rows -->
    <tr>
        <td>
            (Sample input data here)
        </td>
        <td>
            (Sample output data here)
        </td>
    </tr>
    <!-- /sample -->

</table>
<!-- /sample IO table -->

Some programming questions involve inputs or outputs of multiple lines. To add a new line in a HTML data cell, use the <br> tag. Just insert <br> wherever you want a line-break. There is no need to close this tag. The CSS boilerplate contains code to manage the formatting of multi-line cells, so be sure to include it in your code.

<table>

    <!-- CSS boilerplate -->
    <style>
        table {
            width: 100%;
            border-collapse:"separate";
            border-spacing: 4px 8px;
        }

        th {
            text-align: left;
        }

        td {
            background-color: #f5f5f5;
            border: 1px solid black;
            border-radius: 2px;
            vertical-align:top;
            padding: 4px 8px
        }

    </style>
    <!-- /CSS boilerplate -->

    <!-- header -->
    <tr>
        <th>
            Sample input
        </th>
        <th>
            Sample output
        </th>
    </tr>
    <!-- /header -->

    <!-- sample -->
    <tr>
        <td>
            First line of sample input<br>Second line of sample input
        </td>
        <td>
            First line of sample output<br>Second line of sample output<br>Third line of sample output
        </td>
    </tr>
    <!-- /sample -->

//this is a sample config file - copy it to config/config.json
//note that comments in a JSON file are non-standard (so  syntax highlighting is a bit weird, no doubt)
//you can see the context object printed on the console log as you login
{
    "login": true,
    "whitelist":  { "eval": "( hasrole('lecturer') ? ['StandardTheme','Consent','Calculator','GnomeIcons','FreedesktopSounds','editor','Workbook','About-MULE','FileManager','terminal','*'] :['StandardTheme','Consent','Calculator','GnomeIcons','FreedesktopSounds','editor','Workbook','About-MULE','FileManager','terminal'] )" }, //asterisk is a wildcard - allows all applications - just add "*" to the list
    //These are the webdav folders what will be mounted
    "mounts": {
        "home" : {
            "url":{"eval": "default(context.lti.custom_home, 'home')+'/'+context.user.username"}, //this uses a default folder of home/username (unless an lti parameter of custom_home is supplied)
            "access" : "rw" //access can be rw or ro
        },
        "shared": {
            "url":{"eval": "default(context.lti.custom_home, 'home')+'/shared'"}, //this uses a default folder of home/shared
            "access" : { "eval": "( hasrole('lecturer') ? 'rw' : 'ro' )" } //...and is read only for non-lecturers
        },
        "course" : {
            "url": {"eval": "( hasrole('lecturer') ? '/' : null )" }, //this won't appear for non-lecturers (nulls get filtered out)
            "access" : "rw"
        }
    },
    "apps": {
        "workbook": {
            "admin": {"eval": "( hasrole('lecturer'))"}
        }
    }
}

vpl_evaluate.sh

Example 

#! /bin/bash

cat > vpl_execution <<EEOOFF
#! /bin/bash

# ---------- PROGRAMS TESTED (WITHOUT EXTENSION) ---------
prog1= FILENAME
compiled=true

# --------------------- STARTING GRADE -------------------
grade=0

# --------------------- GLOBAL VARIABLES -------------------

declare -a RegexList=('public\s\+static\s\+void\s\+main\s*(\s*String\s*\(args\s*\[\]\|\[\]\s*args\))\s*' 'class\s\+Printing')
declare -a Comment=('created a main method' 'created a class called Printing' )

# --------------------- SETTING VALUES FOR GRADES -------------------
compileGrade=15
numberOfRegex=\${#RegexList[*]}
regexGrade=50
numberOfTestCases=1
testCasesGrade=35
regex=regexGrade/numberOfRegex
testCase=testCasesGrade/numberOfTestCases

# ----------------- COMPILE STUDENT PROG  ----------------
javac \${prog1}.java  &> grepLines.out

#--- if error, assign a mi&imal grade ---
if ((\$? > 0)); then
     echo "------------------------"
     echo "Your program has compiler Errors. Please fix these Errors."
     cat grepLines.out
     echo "------------------------"
     compiled=false
fi

if grep -q '.*' \${prog1}.java
then
    if [ \${compiled} = true ] ; then
        grade=\$((grade+compileGrade))
    fi
fi

# ----------- Remove single line comments from code -------------
sed -i 's|//.*||g' \${prog1}.java


# ----------- TEST THE CODE FOR PARTICULAR PATTERNS -------------
# ----------- TEST Code -------------
c=0
for reg in \${RegexList[*]}; do
    if grep -q \$reg \${prog1}.java
    then
         grade=\$((grade+regex)) #--- adds what % you wanted to give for each regex ---
    else
        echo "------------------------"
        echo "You have not \${Comment[\$c]} in" \${prog1}.java
    fi
    ((c++))
done

cat > data1.txt <<EOF
Welcome to CS620C
EOF

#--- create expected outputs, one for each input file above ---
cat > data1.out <<EOF
Welcome to CS620C
EOF

declare -a AnswersList=('[Ww]elcome\s\+to\s\+[Cc][Ss]161')

if [ \${compiled} = true ] ; then

    c=1
    for ans in \${AnswersList[*]}; do
        echo "------------------------"
        java \${prog1} < data\${c}.txt &> user.out
        if grep -q \$ans user.out
        then
            # Answered Correctly 
            echo ""
            echo "Test Case \${c} Passed"
            #---adds value to grade based on what % you wanted to give for testcases---
            grade=\$((grade+testCase)) 
        else
            # Wrong Answer
            echo "Test Case \${c} Failed"
            #--- display test file ---
            # echo ""
            # echo "Your program tested with:"
            # cat data\${c}.txt

            echo ""
            echo "Your answer:"
            cat user.out

            echo ""
            echo "Expected answer:"
            cat data\${c}.out
        fi
        ((c++))
    done
fi

if (( grade > 100 )); then
    grade=100
fi
if (( grade < 1 )); then
    grade=0
fi

echo ""
echo "------------------------"
echo ""
echo "Grade :=>> \$grade"

EEOOFF

chmod +x vpl_execution

Part C: Troubleshooting

Opening Workbook displays only a blank window

If you edit the lab’s metadata and recompiled the workbook and a blank page appears then you wont be able to recompile the workbook again after you have fixed error in your code. To fix this replace the workbooks.json code with the base code shown in Part B for workbooks.json. This will reset the workbook. Now just recompile the workbook and if all errors are fixed then it should go to the default home page.
